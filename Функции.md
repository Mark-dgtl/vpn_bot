# User

- `create_user(data: NewUserS)` — создать нового пользователя (регистрация).
    
- `get_user_by_id(user_id)` — получить пользователя по PK `id`.
    
- `get_user_by_tg_id(tg_id)` — получить пользователя по Telegram-ID.
    
- `get_user_by_username(username)` — получить пользователя по username.
    
- `update_user(user_id, values)` — частично обновить поля пользователя.
    
- `delete_user(user_id)` — удалить пользователя (и каскадно связанные сущности).
    
- `list_users(filter_opts, page, per_page)` — список пользователей с фильтрами и пагинацией.
    
- `ban_user(user_id, reason)` — пометить как заблокированного.
    
- `unban_user(user_id)` — снять блокировку.
    
- `activate_user(user_id)` — включить `is_active`.
    
- `deactivate_user(user_id)` — выключить `is_active`.
    
- `extend_subscription(user_id, until_datetime)` — продлить подписку (обновить `subscription_end_date`).
    
- `is_subscription_active(user_id)` — вернуть true/false (проверка даты подписки).
    
- `update_last_activity(user_id, timestamp)` — обновить `last_activity`.
    

# Payment

- `create_payment(user_id, amount, screenshot=None)` — создать запись о платеже (статус `pending`).
    
- `get_payment_by_id(payment_id)` — получить платёж по id.
    
- `list_payments(filter_opts, page, per_page)` — получить список платежей (по пользователю, статусу, периоду).
    
- `confirm_payment(payment_id, admin_id)` — пометить платёж `confirmed` (и при необходимости продлить подписку).
    
- `reject_payment(payment_id, reason)` — пометить `rejected`, сохранить причину.
    
- `refund_payment(payment_id, reason)` — пометить `refunded` и логировать возврат.
    
- `update_payment_status(payment_id, status)` — универсальное изменение статуса (с валидацией статусов).
    
- `delete_payment(payment_id)` — удалить платёж (редко, административная опция).
    
- `get_user_payments(user_id)` — все платежи конкретного пользователя.
    
- `aggregate_revenue(period_from, period_to)` — суммарная выручка за период.
    

# Peer (VPN-peer)

- `create_peer(user_id, vpn_ip_address, **kwargs)` — создать peer и зарезервировать IP.
    
- `get_peer_by_id(peer_id)` — получить peer по id.
    
- `get_peer_by_ip(vpn_ip_address)` — найти peer по IP.
    
- `list_user_peers(user_id)` — список peer'ов пользователя.
    
- `update_peer(peer_id, values)` — обновить поля peer (активность, last_handshake и т.д.).
    
- `deactivate_peer(peer_id)` — отключить peer (`is_active=False`).
    
- `activate_peer(peer_id)` — включить peer (`is_active=True`).
    
- `delete_peer(peer_id)` — удалить peer (и связанные traffic_stats).
    
- `assign_ip_to_peer(peer_id, vpn_ip_address)` — назначить/поменять IP (с проверкой уникальности).
    
- `generate_peer_config(peer_id)` — сгенерировать VPN-конфиг для peer.
    
- `revoke_peer_config(peer_id)` — отозвать/удалить конфигурацию.
    

# TrafficStat

- `create_traffic_stat(peer_id, session_start, client_ip)` — создать запись сессии.
    
- `update_traffic_stat(stat_id, bytes_sent, bytes_received, packets_sent, packets_received)` — обновить счётчики.
    
- `close_traffic_session(stat_id, session_end)` — пометить окончание сессии.
    
- `get_traffic_by_peer(peer_id, period_from, period_to)` — список stat'ов по peer и периоду.
    
- `get_active_sessions()` — вернуть все открытые/активные сессии (`session_end IS NULL`).
    
- `aggregate_traffic_for_peer(peer_id, period_from, period_to)` — суммарные байты/пакеты за период.
    
- `aggregate_traffic_for_user(user_id, period_from, period_to)` — суммарные данные по всем peer'ам пользователя.
    
- `delete_traffic_stat(stat_id)` — удалить запись (админская операция).
    
- `prune_old_traffic_stats(older_than_date)` — удалять старые записи (архивация/очистка).
    

# ActivityLog

- `log_activity(user_id, action, description=None, ip_address=None)` — создать запись лога активности.
    
- `get_activity_by_user(user_id, page, per_page)` — получить логи пользователя.
    
- `list_activity(filter_opts, page, per_page)` — общий список логов (фильтр по action/дате/IP).
    
- `delete_activity_log(log_id)` — удалить запись лога.
    
- `prune_old_logs(older_than_date)` — удалять устаревшие логи.
    

# Notification

- `create_notification(user_id, type, title, message)` — создать уведомление для пользователя.
    
- `get_notification_by_id(notification_id)` — получить уведомление.
    
- `list_user_notifications(user_id, unread_only=False, page, per_page)` — список уведомлений пользователя.
    
- `mark_notification_read(notification_id, read_at=datetime.now())` — пометить как прочитанное.
    
- `mark_all_notifications_read(user_id)` — пометить все уведомления пользователя прочитанными.
    
- `delete_notification(notification_id)` — удалить уведомление.
    
- `send_push_notification(user_id, payload)` — (интеграция) отправка реального уведомления через бот/инлайн.
    
- `schedule_notification(send_at_datetime, payload)` — (опционально) отложенная отправка/cron.
    

# Общие/Административные/Утилитарные

- `health_check()` — проверка подключения к базе, ответ сервиса.
    
- `get_stats_overview()` — сводная статистика (кол-во юзеров, активных подписок, доходы).
    
- `export_users_csv(filter_opts)` — экспорт пользователей для админа.
    
- `search_users(query, page, per_page)` — поиск по username/tg_id/first_name.
    
- `backup_database()` — триггер бэкапа БД (если нужно).
    
- `recalculate_user_subscription_from_payments(user_id)` — перестраховочный метод: пересчитать срок подписки по подтверждённым платежам.
    
- `audit_changes(entity, entity_id, old, new, changed_by)` — логирование изменений (если нужна история).
    
- `bulk_create_peers(peers_list)` — массовое создание peer'ов (админская утилита).
    
- `bulk_refund_payments(payment_ids, reason)` — массовый возврат (админ).
